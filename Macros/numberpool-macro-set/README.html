<<numberpool>> macro set for SugarCube v2

SugarCube is a free (gratis and libre) story format for Twine/Twee.

This documentation is a reference for the <<numberpool>> macro set for SugarCube, it assumes at least a passing knowledge of SugarCube macros.

Requirements

This package requires SugarCube v2.5.0 or greater.

Files

The files included in this package are:

  • README.html: This document.
  • numberpool-macro-set.min.js: The, minified, macros.
  • numberpool-macro-set.css: The styles required by the macros.

Installation

JavaScript

  1. Open the numberpool-macro-set.min.js file in a text editor (not a word processor) and copy its contents.
  2. Paste them into your project's script section (Twine 1: script-tagged passage, Twine 2: Story JavaScript).

Stylesheet

  1. Open the numberpool-macro-set.css file in a text editor (not a word processor) and copy its contents.
  2. Paste them into your project's stylesheet section (Twine 1: stylesheet-tagged passage, Twine 2: Story Stylesheet).

Macros

<<numberbox variableName defaultValue minValue maxValue [poolCost] [autofocus]>>

Creates a whole number input box, used to modify the value of the $variable with the given name.

Arguments:

  • variableName: The name of the $variable to modify, which must be quoted (e.g. "$foo"). Object and array property references are also supported (e.g. "$foo.bar", "$foo['bar']", & "$foo[0]").
  • defaultValue: The default value.
  • minValue: The minimum allowed value.
  • maxValue: The maximum allowed value.
  • poolCost: (optional) If part of a number pool, the amount by which the pool is changed for each level of the numberbox value. If omitted, will default to 1. Only useful when part of a pool.
  • autofocus: (optional) Keyword, used to signify that the number box should automatically receive focus. Only use the keyword once per page; attempting to focus more than one element is undefined behavior.

Usage:

→ Creates a number box which modifies $Strength
<<numberbox "$Strength" $Strength 1 20>>

→ Creates an automatically focused number box which modifies $Strength
<<numberbox "$Strength" $Strength 1 20 autofocus>>

<<numberpool variableName>>

Creates a number pool for use with the other macros from this set. Only useful when paired with them.

Arguments:

  • variableName: The name of the $variable to modify, which must be quoted (e.g. "$foo"). Object and array property references are also supported (e.g. "$foo.bar", "$foo['bar']", & "$foo[0]").

Usage:

<<numberpool "$StatPool">>
|''ST:''|<<numberbox "$ST" $ST 1 20>>|
|''DX:''|<<numberbox "$DX" $DX 1 20>>|
|''IQ:''|<<numberbox "$IQ" $IQ 1 20>>|
|''HT:''|<<numberbox "$HT" $HT 1 20>>|
<</numberpool>>

<<onchange>>

Silently executes its contents when the pool is changed via one of the other macros from this set.

Arguments: none

Usage:

You have <span id="stat-pool">$StatPool</span> points remaining to allocate.
<<numberpool "$StatPool">>
|''ST:''|<<numberbox "$ST" $ST 1 20>>|
|''DX:''|<<numberbox "$DX" $DX 1 20>>|
|''IQ:''|<<numberbox "$IQ" $IQ 1 20>>|
|''HT:''|<<numberbox "$HT" $HT 1 20>>|
<<onchange>>
    <<replace "#stat-pool">>$StatPool<</replace>>
<</numberpool>>

<</numberpool>>

Closes the <<numberpool>> macro.

<<numberslider variableName defaultValue minValue maxValue stepValue [autofocus]>>

Creates a number range slider, used to modify the value of the $variable with the given name.

Arguments:

  • variableName: The name of the $variable to modify, which must be quoted (e.g. "$foo"). Object and array property references are also supported (e.g. "$foo.bar", "$foo['bar']", & "$foo[0]").
  • defaultValue: The default value. It should be a multiple of the step value plus the minimum value.
  • minValue: The minimum allowed value.
  • maxValue: The maximum allowed value. It must be a multiple of the step value plus the minimum value.
  • stepValue: The allowed increments to which a value may be set.
  • autofocus: (optional) Keyword, used to signify that the number slider should automatically receive focus. Only use the keyword once per page; attempting to focus more than one element is undefined behavior.

Usage:

→ Creates a number slider which modifies $opacity
<<numberslider "$opacity" $opacity 0 1 0.5>>

→ Creates an automatically focused number slider which modifies $opacity
<<numberslider "$opacity" $opacity 0 1 0.5 autofocus>>
